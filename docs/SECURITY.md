# Security Policy / Política de Segurança

## EN
This project prioritizes security in network operations. We follow best practices to ensure that chaos experiments do not compromise the integrity of the host system.

### Reporting Vulnerabilities
If you discover a security vulnerability in this project, please report it via GitHub Issues or contact me directly via LinkedIn.

### Security Measures
- **Least Privilege:** Chaos experiments are executed only on specific network interfaces.
- **Controlled Environment:** All experiments are conducted in isolated network labs to prevent impact on production traffic.
- **Audit Logs:** All modifications to network configurations are logged in `results/`.

## 🇧🇷 Português
### Política de Segurança
Este projeto prioriza a segurança em operações de rede. Seguimos as melhores práticas para garantir que experimentos de caos não comprometam a integridade do sistema host.

### Relatando Vulnerabilidades
Se você descobrir uma vulnerabilidade de segurança neste projeto, reporte-a através das GitHub Issues ou entre em contato diretamente via LinkedIn.

### Medidas de Segurança
- **Menor Privilégio:** Experimentos de caos são executados apenas em interfaces de rede específicas.
- **Ambiente Controlado:** Todos os experimentos são conduzidos em laboratórios de rede isolados para prevenir impactos no tráfego de produção.
- **Logs de Auditoria:** Todas as modificações nas configurações de rede são registradas na pasta `results/`.
